<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| in-memory pooled matrix → per-image .qpdata writes | Pass 2 mutates and saves each image's ImageData; a wrong/partial write corrupts persisted analysis |
| pass-1 read ↔ pass-2 re-read of the same image | A user editing/deleting cells between passes changes the cell set |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-07 | Tampering (data integrity) | Partial/corrupt Cluster write on failure or mid-run cancel | mitigate | Cluster is a NON-DESTRUCTIVE numeric measurement (phenotype getPathClass never touched); cancel is honored between images leaving already-written images intact and unwritten ones untouched; per-image save wrapped in try/catch that logs and continues (D-12). No rollback needed. |
| T-15-08 | Tampering (accidental) | Cell edited/deleted between pass 1 and pass 2 → UUID mismatch | accept | Pass-2 UUID lookup returning null → cell silently skipped (Cluster = -1.0), never a crash (RESEARCH Security note). Natural Map.get(null) behaviour. |
| T-15-09 | Denial of Service | Pooling every cell holding all hierarchies in memory | mitigate | Two-pass design: pass 1 releases each hierarchy before the next; only the pooled primitive rows + packed-long UUIDs are retained (RESEARCH memory-safety principle). Soft ceiling enforced in Plan 05. |
| T-15-10 | Tampering (integrity of result) | Recall-gate failure still writing labels | mitigate | AnnRecallException from clusterViaAnn propagates BEFORE pass 2 begins → zero images written (aborted result). Task 3 Test D. |
</threat_model>
