<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Saved prediction counts -> analyzer inputs | Untrusted/skewed per-image counts can produce false anomaly labels if not normalized |
| Analyzer outputs -> UI decision support | Mis-scored rows may cause users to focus on wrong images/classes |
| Threshold constants -> operational workflow | Overly sensitive defaults can create alert fatigue and hide true outliers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | T | CohortAnomalyAnalyzer inputs | mitigate | Normalize by proportions, apply minimum count gates, and use smoothing to prevent sparse-count distortion |
| T-11-02 | I | Rare-enrichment reporting | mitigate | Emit per-class counts and enrichment folds together so users can inspect evidence behind flags |
| T-11-03 | D | Outlier thresholding | mitigate | Use robust z-score (median/MAD) and deterministic tests to prevent unstable anomaly churn |
| T-11-04 | R | Scoring regressions | mitigate | Add dedicated unit tests asserting ranking and reason tags on fixed synthetic datasets |
</threat_model>
