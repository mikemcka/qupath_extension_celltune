build (single-image preview, all-cells cohort driver, and the Leiden kNN-transfer query path),
closing the third and last documented scanpy-fidelity gap ("no PCA before neighbours"; the other
two — CPM vs modularity, SNN/Jaccard vs UMAP connectivities — remain out of scope, unchanged from
Phase 14/15). Mirrors scanpy's `scale -> PCA(50) -> neighbors -> leiden` recipe: a no-op below a
feature-count threshold (small curated marker panels), otherwise an exact deterministic Smile PCA
projection, fit on a bounded seeded subsample at cohort scale and applied to every row.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user project data → in-JVM PCA projection | Marker vectors (user's own QuPath data) reduced in-memory via Smile PCA; no network, no external/untrusted input |
| all-cells cohort fit → bounded subsample | Projection is fit on a capped, seeded subsample rather than the full pooled matrix, bounding memory independent of total cell count |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-01 | Denial of Service | PCA fit over an unbounded pooled matrix at cohort scale | mitigate | `fitSampleCap` (default 100k) bounds the FIT set; the projection is then applied (cheap, streaming-safe) to every pooled row. |
| T-16-02 | Tampering (data integrity) | Cluster centroids silently computed in the wrong (PCA) space, corrupting the interpretive marker-space heatmap | mitigate | `centroidsAndCounts` is always called against the original marker-space matrix, never the PCA-reduced one; pinned by a dedicated regression test. |
</threat_model>
