<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User-selected bundle path -> filesystem write/read | User controls source and destination paths, including stale or tampered files |
| UI import mode selection -> data mutation path | Mode choice directly controls overwrite behavior in project state |
| Import report -> user trust | Inaccurate summary can mislead users about what data was changed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-05 | T | Bundle file selection | mitigate | Require explicit project-open checks and reject unreadable/non-bundle files with clear error dialogs |
| T-12-06 | R | Import mode semantics | mitigate | Force explicit merge/replace choice before import; include mode in success summary text |
| T-12-07 | I | User-facing import outcomes | mitigate | Display marker-level imported/replaced/skipped/failed counts from IO report object |
| T-12-08 | D | Repeated import attempts | mitigate | Keep import idempotent per mode and provide deterministic skip reasons for incompatible marker schemas |
</threat_model>
