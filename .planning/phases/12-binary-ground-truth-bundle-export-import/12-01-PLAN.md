<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Bundle file on disk -> import parser | User-selected files are untrusted and may be malformed or intentionally tampered |
| Manifest marker IDs -> project state files | Marker names cross from bundle metadata into filesystem-backed binary state |
| Imported training rows -> model training inputs | Corrupted rows can poison classifier behavior if not validated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | T | Bundle manifest / payload parsing | mitigate | Validate schema version, required keys, and marker payload shape before applying writes |
| T-12-02 | T | Marker name to file path mapping | mitigate | Re-sanitize every imported marker via `BinaryClassifierRegistry.sanitizeMarkerName` and reject invalid names |
| T-12-03 | D | Importing large malformed bundles | mitigate | Parse marker payloads one marker at a time and skip invalid entries without aborting entire import |
| T-12-04 | R | Silent overwrite of existing training rows | mitigate | Require explicit import mode (`MERGE` or `REPLACE`) and emit marker-level result report with counts |
</threat_model>
