<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| UI rule text -> parser | User-provided expression text is untrusted |
| persisted rules -> classifier | Stored rules can be stale/malformed and must be validated before use |
| classifier output -> PathClass assignment | Incorrect rule evaluation can mislabel detections |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | T | rule expression parsing | mitigate | Strict parser with explicit polarity tokens and duplicate-marker rejection |
| T-03-02-02 | D | batch rule application | mitigate | Bound marker/rule sizes and skip invalid rules without crashing batch execution |
| T-03-02-03 | R | rule application auditability | mitigate | Log rule name/expression used for each apply and batch result line |
</threat_model>
